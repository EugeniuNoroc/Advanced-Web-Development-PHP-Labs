# Лабораторная работа №6. Обработка и валидация форм

## Цель работы

Освоить основные принципы работы с HTML-формами в PHP, включая отправку данных на сервер и их обработку, включая валидацию данных. Реализовать решение с использованием объектно-ориентированного программирования.

---

## Тема проекта

**SkyLog — Дневник наблюдений за звёздным небом.**

Приложение позволяет вести учёт астрономических наблюдений: добавлять записи о наблюдаемых объектах, редактировать их и удалять. Список наблюдений отображается в виде таблицы с возможностью сортировки по любому полю.

---

## Шаг 1. Определение модели данных

Для темы «Дневник наблюдений за звёздным небом» определена следующая модель данных:

| Поле             | Тип    | Описание                                              |
|------------------|--------|-------------------------------------------------------|
| `id`             | string | Уникальный идентификатор записи                       |
| `object_name`    | string | Название наблюдаемого объекта                         |
| `object_type`    | enum   | Тип объекта: `planet`, `star`, `galaxy`, `nebula`, `meteor`, `other` |
| `date`           | date   | Дата наблюдения (YYYY-MM-DD)                          |
| `time`           | string | Время начала наблюдения (HH:MM)                       |
| `conditions`     | enum   | Условия наблюдения: `excellent`, `good`, `fair`, `poor` |
| `description`    | text   | Подробное описание наблюдения                         |
| `used_telescope` | bool   | Флаг использования телескопа или бинокля (checkbox)   |
| `created_at`     | string | Дата и время создания записи (заполняется автоматически) |

Модель удовлетворяет всем требованиям: присутствует минимум 6 полей, поля типа `string`, `date`, `enum` и `text`.

Приложение состоит из четырёх файлов, каждый из которых выполняет одну задачу:

## Шаг 3. ООП-реализация

### Класс `Observation`

Модель данных одного наблюдения. Содержит все поля и два статических фабричных метода для преобразования данных.

```php
/**
 * Модель одного наблюдения за небом.
 */
class Observation
{
    /** @var string Уникальный идентификатор */
    public string $id;

    /** @var string Название объекта (например, "Орион") */
    public string $object_name;

    /** @var string Тип объекта: planet|star|galaxy|nebula|meteor|other */
    public string $object_type;

    /** @var string Дата наблюдения (YYYY-MM-DD) */
    public string $date;

    /** @var string Время наблюдения (HH:MM) */
    public string $time;

    /** @var string Условия наблюдения: excellent|good|fair|poor */
    public string $conditions;

    /** @var string Описание — что именно наблюдалось */
    public string $description;

    /** @var bool Использовался ли телескоп */
    public bool $used_telescope;

    /** @var string Дата создания записи */
    public string $created_at;

    /**
     * Создаёт объект из массива данных (например, из $_POST или из файла).
     *
     * @param array $data Ассоциативный массив с данными наблюдения
     * @return self
     */
    public static function fromArray(array $data): self
    {
        $obs = new self();
        $obs->id             = $data['id']          ?? uniqid();
        $obs->object_name    = trim($data['object_name']  ?? '');
        $obs->object_type    = $data['object_type']  ?? 'other';
        $obs->date           = $data['date']         ?? '';
        $obs->time           = $data['time']         ?? '';
        $obs->conditions     = $data['conditions']   ?? 'good';
        $obs->description    = trim($data['description']  ?? '');
        $obs->used_telescope = !empty($data['used_telescope']);
        $obs->created_at     = $data['created_at']   ?? date('Y-m-d H:i:s');
        return $obs;
    }

    /**
     * Конвертирует объект в массив для сохранения в JSON.
     *
     * @return array
     */
    public function toArray(): array
    {
        return [
            'id'             => $this->id,
            'object_name'    => $this->object_name,
            'object_type'    => $this->object_type,
            'date'           => $this->date,
            'time'           => $this->time,
            'conditions'     => $this->conditions,
            'description'    => $this->description,
            'used_telescope' => $this->used_telescope,
            'created_at'     => $this->created_at,
        ];
    }
}
```

### Класс `ObservationValidator`

Отвечает исключительно за валидацию данных. Принимает массив из `$_POST`, возвращает список ошибок. Если список пустой — данные корректны.

```php
/**
 * Валидатор данных наблюдения.
 * Возвращает список ошибок; если список пустой — данные корректны.
 */
class ObservationValidator
{
    /** @var string[] Допустимые типы объектов */
    private const VALID_TYPES = ['planet', 'star', 'galaxy', 'nebula', 'meteor', 'other'];

    /** @var string[] Допустимые условия наблюдения */
    private const VALID_CONDITIONS = ['excellent', 'good', 'fair', 'poor'];

    /**
     * Проверяет массив данных и возвращает массив ошибок.
     *
     * @param array $data Данные из формы
     * @return string[] Массив сообщений об ошибках (пустой, если ошибок нет)
     */
    public function validate(array $data): array
    {
        $errors = [];

        if (empty(trim($data['object_name'] ?? ''))) {
            $errors[] = 'Укажите название объекта.';
        } elseif (strlen($data['object_name']) > 100) {
            $errors[] = 'Название не должно превышать 100 символов.';
        }

        if (!in_array($data['object_type'] ?? '', self::VALID_TYPES, true)) {
            $errors[] = 'Выберите допустимый тип объекта.';
        }

        if (empty($data['date'] ?? '')) {
            $errors[] = 'Укажите дату наблюдения.';
        } elseif (!$this->isValidDate($data['date'])) {
            $errors[] = 'Дата имеет неверный формат.';
        }

        if (empty($data['time'] ?? '')) {
            $errors[] = 'Укажите время наблюдения.';
        }

        if (!in_array($data['conditions'] ?? '', self::VALID_CONDITIONS, true)) {
            $errors[] = 'Выберите допустимые условия наблюдения.';
        }

        if (empty(trim($data['description'] ?? ''))) {
            $errors[] = 'Добавьте описание наблюдения.';
        } elseif (strlen($data['description']) < 10) {
            $errors[] = 'Описание слишком короткое (минимум 10 символов).';
        }

        return $errors;
    }

    /**
     * Проверяет, является ли строка корректной датой формата YYYY-MM-DD.
     *
     * @param string $date Строка с датой
     * @return bool
     */
    private function isValidDate(string $date): bool
    {
        $d = DateTime::createFromFormat('Y-m-d', $date);
        return $d && $d->format('Y-m-d') === $date;
    }
}
```

### Класс `ObservationStorage`

Отвечает за чтение и запись данных в JSON-файл. Реализует полный набор операций: загрузка всех записей, добавление, обновление, удаление и поиск по ID.

```php
/**
 * Хранилище наблюдений: читает и пишет JSON-файл.
 */
class ObservationStorage
{
    /** @var string Путь к файлу с данными */
    private string $file;

    /**
     * @param string $file Путь к JSON-файлу
     */
    public function __construct(string $file = 'data.json')
    {
        $this->file = $file;
    }

    /**
     * Загружает все наблюдения из файла.
     *
     * @return Observation[]
     */
    public function loadAll(): array
    {
        if (!file_exists($this->file)) {
            return [];
        }
        $raw = json_decode(file_get_contents($this->file), true) ?? [];
        return array_map(fn($item) => Observation::fromArray($item), $raw);
    }

    /**
     * Сохраняет список наблюдений в файл.
     *
     * @param Observation[] $observations
     * @return void
     */
    public function saveAll(array $observations): void
    {
        $data = array_map(fn($obs) => $obs->toArray(), $observations);
        file_put_contents(
            $this->file,
            json_encode(array_values($data), JSON_PRETTY_PRINT | JSON_UNESCAPED_UNICODE)
        );
    }

    /**
     * Добавляет одно наблюдение и сохраняет файл.
     *
     * @param Observation $observation
     * @return void
     */
    public function add(Observation $observation): void
    {
        $all   = $this->loadAll();
        $all[] = $observation;
        $this->saveAll($all);
    }

    /**
     * Обновляет существующее наблюдение по ID.
     *
     * @param Observation $updated Обновлённый объект
     * @return bool true если найдено и обновлено
     */
    public function update(Observation $updated): bool
    {
        $all = $this->loadAll();
        foreach ($all as $i => $obs) {
            if ($obs->id === $updated->id) {
                $updated->created_at = $obs->created_at;
                $all[$i] = $updated;
                $this->saveAll($all);
                return true;
            }
        }
        return false;
    }

    /**
     * Удаляет наблюдение по ID.
     *
     * @param string $id
     * @return bool true если найдено и удалено
     */
    public function delete(string $id): bool
    {
        $all      = $this->loadAll();
        $filtered = array_filter($all, fn($obs) => $obs->id !== $id);
        if (count($filtered) === count($all)) {
            return false;
        }
        $this->saveAll($filtered);
        return true;
    }

    /**
     * Находит наблюдение по ID.
     *
     * @param string $id
     * @return Observation|null
     */
    public function findById(string $id): ?Observation
    {
        foreach ($this->loadAll() as $obs) {
            if ($obs->id === $id) return $obs;
        }
        return null;
    }
}
```

## Шаг 4. Создание HTML-формы и обработка данных на сервере

Файл `form.php` выполняет сразу две роли: отображает форму (для создания и для редактирования) и обрабатывает её отправку. Режим определяется наличием параметра `id` в запросе.

```php
session_start();
require_once 'classes.php';

$storage   = new ObservationStorage('data.json');
$validator = new ObservationValidator();

$editId = $_GET['id'] ?? $_POST['id'] ?? null;
$isEdit = $editId !== null;
$errors = [];

// При редактировании — предзаполняем форму данными из файла
if ($isEdit && $_SERVER['REQUEST_METHOD'] !== 'POST') {
    $existing = $storage->findById($editId);
    if (!$existing) {
        header('Location: index.php');
        exit;
    }
    $data = $existing->toArray();
} else {
    $data = $_POST ?: [];
}

// Обработка отправки формы
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $errors = $validator->validate($data);

    if (empty($errors)) {
        $obs = Observation::fromArray($data);

        if ($isEdit) {
            $storage->update($obs);
            $_SESSION['flash'] = ['type' => 'success', 'msg' => 'Наблюдение обновлено!'];
        } else {
            $storage->add($obs);
            $_SESSION['flash'] = ['type' => 'success', 'msg' => 'Наблюдение добавлено!'];
        }

        header('Location: index.php');
        exit;
    }
}
```

Форма отправляется методом `POST`. Клиентская валидация реализована через атрибуты `required`, `maxlength` и `minlength`. Серверная валидация выполняется классом `ObservationValidator`. При ошибках форма отображается повторно с сохранёнными введёнными данными и списком ошибок.

Два вспомогательных хелпера обеспечивают безопасный вывод значений в HTML:

```php
/** Безопасно возвращает значение поля для вывода в HTML */
function val(array $data, string $key): string {
    return htmlspecialchars($data[$key] ?? '');
}

/** Проверяет, выбрано ли значение select */
function sel(array $data, string $key, string $value): string {
    return ($data[$key] ?? '') === $value ? 'selected' : '';
}
```

## Шаг 5. Вывод данных и сортировка

Файл `index.php` загружает все наблюдения из файла и сортирует их по выбранному полю. Параметры сортировки передаются через GET-запрос (`?sort=date&dir=asc`). Допустимые значения проверяются белым списком, чтобы исключить подстановку произвольных имён полей.

```php
$sortField    = $_GET['sort'] ?? 'date';
$sortDir      = $_GET['dir']  ?? 'desc';
$allowedSorts = ['date', 'object_name', 'object_type', 'conditions', 'created_at'];

if (!in_array($sortField, $allowedSorts, true)) $sortField = 'date';
if (!in_array($sortDir, ['asc', 'desc'], true)) $sortDir = 'desc';

usort($observations, function ($a, $b) use ($sortField, $sortDir) {
    $cmp = strcmp($a->$sortField, $b->$sortField);
    return $sortDir === 'asc' ? $cmp : -$cmp;
});
```

Клик по заголовку столбца переключает направление сортировки. Текущее направление отображается стрелкой рядом с названием столбца.

## Шаг 6. Удаление записи

Файл `delete.php` принимает `id` через GET-параметр, вызывает `$storage->delete($id)` и перенаправляет обратно на главную страницу с flash-сообщением о результате.

```php
session_start();
require_once 'classes.php';

$id = $_GET['id'] ?? null;

if (!$id) {
    header('Location: index.php');
    exit;
}

$storage = new ObservationStorage('data.json');
$deleted = $storage->delete($id);

$_SESSION['flash'] = $deleted
    ? ['type' => 'success', 'msg' => 'Наблюдение удалено.']
    : ['type' => 'error',   'msg' => 'Запись не найдена.'];

header('Location: index.php');
exit;
```

Перед удалением в браузере отображается стандартный диалог подтверждения (`confirm()`), чтобы исключить случайное удаление.

## Контрольные вопросы

### 1. Какие существуют методы отправки данных из формы на сервер? Какие методы поддерживает HTML-форма?

HTML-форма поддерживает два метода, задаваемых атрибутом `method`:

- **GET** — данные передаются в строке URL в виде параметров запроса (`?name=value`). Подходит для поисковых запросов и фильтрации, так как результат можно сохранить в закладки. Не подходит для передачи чувствительных данных или больших объёмов информации, так как URL ограничен по длине и виден в адресной строке.
- **POST** — данные передаются в теле HTTP-запроса, не отображаются в URL. Используется для создания и изменения данных, отправки паролей и загрузки файлов.

В данной работе форма использует метод `POST`, так как данные изменяют состояние приложения (создают или обновляют запись).

---

### 2. Какие глобальные переменные используются для доступа к данным формы в PHP?

PHP предоставляет несколько суперглобальных массивов для доступа к данным запроса:

- **`$_POST`** — содержит данные, отправленные методом POST. Используется в `form.php` для получения значений всех полей формы.
- **`$_GET`** — содержит параметры из строки URL. Используется в `index.php` для получения параметров сортировки (`sort`, `dir`) и в `delete.php` / `form.php` для получения `id`.
- **`$_REQUEST`** — объединяет `$_GET`, `$_POST` и `$_COOKIE`. Удобен, но менее явен, поэтому в данной работе не применяется.
- **`$_SESSION`** — используется для передачи flash-сообщений между страницами после редиректа.
- **`$_FILES`** — используется при загрузке файлов (в данной работе не применяется).

---

### 3. Как обеспечить безопасность при обработке данных из формы (например, защититься от XSS)?

**XSS (Cross-Site Scripting)** — атака, при которой злоумышленник внедряет вредоносный JavaScript в страницу через пользовательский ввод. Для защиты применяются следующие меры:

- **Экранирование вывода через `htmlspecialchars()`** — все данные перед выводом в HTML проходят через эту функцию. Она заменяет символы `<`, `>`, `"`, `'`, `&` на соответствующие HTML-сущности, не позволяя браузеру интерпретировать их как код. В данной работе для этого создан хелпер `val()`, который применяется ко всем полям формы и ячейкам таблицы.

```php
function val(array $data, string $key): string {
    return htmlspecialchars($data[$key] ?? '');
}
```

- **Серверная валидация** — данные проверяются по белым спискам допустимых значений (`VALID_TYPES`, `VALID_CONDITIONS`). Значение, не входящее в список, отклоняется с ошибкой, даже если клиентская валидация была обойдена.

- **Никогда не доверять клиентской валидации** — атрибуты `required`, `maxlength` и `minlength` на HTML-элементах улучшают пользовательский опыт, но легко обходятся. Вся критическая проверка выполняется повторно на сервере в классе `ObservationValidator`.
