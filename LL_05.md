## Задание 2. Класс Transaction

Создайте класс `Transaction`, описывающий одну банковскую транзакцию.

```php
<?php

/**
 * Класс, представляющий одну банковскую транзакцию.
 */
class Transaction
{
    /**
     * @param int    $id          Уникальный идентификатор транзакции.
     * @param string $date        Дата транзакции в формате YYYY-MM-DD.
     * @param float  $amount      Сумма транзакции.
     * @param string $description Описание платежа.
     * @param string $merchant    Получатель платежа.
     */
    public function __construct(
        private int    $id,
        private string $date,
        private float  $amount,
        private string $description,
        private string $merchant,
    ) {}

    /** @return int Уникальный идентификатор транзакции. */
    public function getId(): int
    {
        return $this->id;
    }

    /** @return string Дата транзакции. */
    public function getDate(): string
    {
        return $this->date;
    }

    /** @return float Сумма транзакции. */
    public function getAmount(): float
    {
        return $this->amount;
    }

    /** @return string Описание платежа. */
    public function getDescription(): string
    {
        return $this->description;
    }

    /** @return string Получатель платежа. */
    public function getMerchant(): string
    {
        return $this->merchant;
    }

    /**
     * Возвращает количество дней с момента транзакции до текущей даты.
     *
     * @return int Количество дней.
     */
    public function getDaysSinceTransaction(): int
    {
        $transactionDate = new DateTime($this->date);
        $today           = new DateTime('today');
        $diff            = $today->diff($transactionDate);
        return (int) $diff->days;
    }
}
```

## Задание 3. Интерфейс TransactionStorageInterface

Интерфейс определяет контракт для любого хранилища транзакций.

```php
<?php

/**
 * Интерфейс для хранилища транзакций.
 * Определяет базовые операции доступа к коллекции транзакций.
 */
interface TransactionStorageInterface
{
    /**
     * Добавляет новую транзакцию в хранилище.
     *
     * @param Transaction $transaction Объект транзакции.
     * @return void
     */
    public function addTransaction(Transaction $transaction): void;

    /**
     * Удаляет транзакцию по идентификатору.
     *
     * @param int $id Идентификатор транзакции.
     * @return void
     */
    public function removeTransactionById(int $id): void;

    /**
     * Возвращает список всех транзакций.
     *
     * @return Transaction[]
     */
    public function getAllTransactions(): array;

    /**
     * Ищет транзакцию по идентификатору.
     *
     * @param int $id Идентификатор транзакции.
     * @return Transaction|null Найденная транзакция или null.
     */
    public function findById(int $id): ?Transaction;
}
```

## Задание 4. Класс TransactionRepository

Класс реализует интерфейс `TransactionStorageInterface` и отвечает за хранение транзакций.

```php
<?php

/**
 * Репозиторий для хранения и управления коллекцией транзакций.
 * Реализует интерфейс TransactionStorageInterface.
 */
class TransactionRepository implements TransactionStorageInterface
{
    /**
     * Массив транзакций.
     *
     * @var Transaction[]
     */
    private array $transactions = [];

    /**
     * Добавляет новую транзакцию в репозиторий.
     *
     * @param Transaction $transaction Объект транзакции.
     * @return void
     */
    public function addTransaction(Transaction $transaction): void
    {
        $this->transactions[] = $transaction;
    }

    /**
     * Удаляет транзакцию по идентификатору.
     *
     * @param int $id Идентификатор транзакции.
     * @return void
     */
    public function removeTransactionById(int $id): void
    {
        $this->transactions = array_values(
            array_filter(
                $this->transactions,
                fn(Transaction $t): bool => $t->getId() !== $id
            )
        );
    }

    /**
     * Возвращает список всех транзакций.
     *
     * @return Transaction[]
     */
    public function getAllTransactions(): array
    {
        return $this->transactions;
    }

    /**
     * Ищет транзакцию по идентификатору.
     *
     * @param int $id Идентификатор транзакции.
     * @return Transaction|null Найденная транзакция или null.
     */
    public function findById(int $id): ?Transaction
    {
        foreach ($this->transactions as $transaction) {
            if ($transaction->getId() === $id) {
                return $transaction;
            }
        }
        return null;
    }
}
```

## Задание 5. Класс TransactionManager

Класс реализует бизнес-логику, опираясь на интерфейс хранилища.

```php
<?php

/**
 * Менеджер транзакций.
 * Содержит бизнес-логику для работы с коллекцией транзакций.
 * Не хранит транзакции самостоятельно — делегирует это репозиторию.
 */
class TransactionManager
{
    /**
     * @param TransactionStorageInterface $repository Хранилище транзакций.
     */
    public function __construct(
        private TransactionStorageInterface $repository
    ) {}

    /**
     * Вычисляет общую сумму всех транзакций.
     *
     * @return float Общая сумма.
     */
    public function calculateTotalAmount(): float
    {
        $total = 0.0;
        foreach ($this->repository->getAllTransactions() as $transaction) {
            $total += $transaction->getAmount();
        }
        return $total;
    }

    /**
     * Вычисляет сумму транзакций за указанный период.
     *
     * @param string $startDate Начало периода (YYYY-MM-DD).
     * @param string $endDate   Конец периода (YYYY-MM-DD).
     * @return float Сумма транзакций за период.
     */
    public function calculateTotalAmountByDateRange(string $startDate, string $endDate): float
    {
        $start = new DateTime($startDate);
        $end   = new DateTime($endDate);
        $total = 0.0;

        foreach ($this->repository->getAllTransactions() as $transaction) {
            $date = new DateTime($transaction->getDate());
            if ($date >= $start && $date <= $end) {
                $total += $transaction->getAmount();
            }
        }

        return $total;
    }

    /**
     * Подсчитывает количество транзакций по определённому получателю.
     *
     * @param string $merchant Название получателя.
     * @return int Количество транзакций.
     */
    public function countTransactionsByMerchant(string $merchant): int
    {
        $count = 0;
        foreach ($this->repository->getAllTransactions() as $transaction) {
            if (strcasecmp($transaction->getMerchant(), $merchant) === 0) {
                $count++;
            }
        }
        return $count;
    }

    /**
     * Возвращает транзакции, отсортированные по дате (по возрастанию).
     *
     * @return Transaction[]
     */
    public function sortTransactionsByDate(): array
    {
        $transactions = $this->repository->getAllTransactions();
        usort(
            $transactions,
            fn(Transaction $a, Transaction $b): int =>
                new DateTime($a->getDate()) <=> new DateTime($b->getDate())
        );
        return $transactions;
    }

    /**
     * Возвращает транзакции, отсортированные по сумме (по убыванию).
     *
     * @return Transaction[]
     */
    public function sortTransactionsByAmountDesc(): array
    {
        $transactions = $this->repository->getAllTransactions();
        usort(
            $transactions,
            fn(Transaction $a, Transaction $b): int =>
                $b->getAmount() <=> $a->getAmount()
        );
        return $transactions;
    }
}
```

## Задание 6. Класс TransactionTableRenderer

Класс отвечает исключительно за формирование HTML-таблицы транзакций.

```php
<?php

/**
 * Рендерер таблицы транзакций в HTML.
 * Отвечает только за представление данных — не содержит бизнес-логики.
 */
final class TransactionTableRenderer
{
    /**
     * Принимает массив транзакций и возвращает HTML-код таблицы.
     *
     * @param Transaction[] $transactions Массив транзакций для отображения.
     * @return string HTML-строка с таблицей транзакций.
     */
    public function render(array $transactions): string
    {
        $html  = '<table border="1" cellpadding="8" cellspacing="0">';
        $html .= '<thead><tr>';
        $html .= '<th>ID</th>';
        $html .= '<th>Дата</th>';
        $html .= '<th>Сумма</th>';
        $html .= '<th>Описание</th>';
        $html .= '<th>Получатель</th>';
        $html .= '<th>Дней назад</th>';
        $html .= '</tr></thead>';
        $html .= '<tbody>';

        foreach ($transactions as $transaction) {
            $html .= '<tr>';
            $html .= '<td>' . htmlspecialchars((string) $transaction->getId())          . '</td>';
            $html .= '<td>' . htmlspecialchars($transaction->getDate())                 . '</td>';
            $html .= '<td>' . number_format($transaction->getAmount(), 2)               . '</td>';
            $html .= '<td>' . htmlspecialchars($transaction->getDescription())          . '</td>';
            $html .= '<td>' . htmlspecialchars($transaction->getMerchant())             . '</td>';
            $html .= '<td>' . $transaction->getDaysSinceTransaction()                   . '</td>';
            $html .= '</tr>';
        }

        $html .= '</tbody></table>';
        return $html;
    }
}
```

## Контрольные вопросы

### 1. Зачем нужна строгая типизация в PHP и как она помогает при разработке?

`declare(strict_types=1)` включает режим, при котором PHP не выполняет неявное приведение типов при вызове функций и методов. Например, если метод ожидает `int`, а передана строка `"5"` — будет выброшен `TypeError`. Это позволяет выявлять ошибки раньше, делает код более предсказуемым и снижает количество трудноуловимых багов, связанных с автоматическим преобразованием типов.

---

### 2. Что такое класс в ООП и какие основные компоненты класса вы знаете?

Класс — это шаблон (blueprint) для создания объектов, описывающий их состояние и поведение. Основные компоненты класса:

- **Свойства (поля)** — переменные, хранящие состояние объекта (`private int $id`).
- **Конструктор** — специальный метод `__construct()`, вызываемый при создании объекта.
- **Методы** — функции, определяющие поведение объекта (`getAmount()`, `getDaysSinceTransaction()`).
- **Модификаторы доступа** — `public`, `protected`, `private` — управляют видимостью членов класса.

---

### 3. Объясните, что такое полиморфизм и как он может быть реализован в PHP.

Полиморфизм — это способность объектов разных классов реагировать на одинаковые вызовы методов по-разному. В PHP он реализуется двумя основными способами:

- **Через интерфейсы** — несколько классов реализуют один интерфейс, но каждый по-своему. Например, `TransactionRepository` и гипотетический `DatabaseTransactionRepository` оба реализуют `TransactionStorageInterface`, но один хранит данные в памяти, а другой — в базе данных.
- **Через наследование** — дочерний класс переопределяет метод родителя (`override`).

```php
// Пример полиморфизма через интерфейс:
function printCount(TransactionStorageInterface $storage): void {
    echo count($storage->getAllTransactions());
}
// Работает с любой реализацией интерфейса
```

### 4. Что такое интерфейс в PHP и как он отличается от абстрактного класса?

**Интерфейс** — это контракт, описывающий набор методов без их реализации. Класс, реализующий интерфейс, обязан определить все его методы.

**Абстрактный класс** — это класс, который может содержать как абстрактные методы (без реализации), так и обычные (с реализацией). От него нельзя создать объект напрямую.

Ключевые отличия:

| Критерий | Интерфейс | Абстрактный класс |
|---|---|---|
| Реализация методов | Нет | Может быть |
| Свойства | Только константы | Любые |
| Наследование | Класс может реализовывать несколько | Только один |
| Конструктор | Нет | Может быть |

### 5. Какие преимущества даёт использование интерфейсов при проектировании архитектуры приложения?

Использование `TransactionStorageInterface` в данной лабораторной работе даёт следующие преимущества:

- **Слабая связанность** — `TransactionManager` зависит от интерфейса, а не от конкретного класса. Это означает, что его код не нужно менять при замене реализации хранилища.
- **Заменяемость реализаций** — можно создать `DatabaseRepository`, `JsonFileRepository` или `InMemoryRepository`, и `TransactionManager` будет работать с любым из них без изменений.
- **Тестируемость** — в тестах легко передать mock-объект, реализующий интерфейс, вместо реального репозитория.
- **Явный контракт** — интерфейс документирует, какие операции обязан поддерживать любой репозиторий, что упрощает понимание архитектуры для других разработчиков.
